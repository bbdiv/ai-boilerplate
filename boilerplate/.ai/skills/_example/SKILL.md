# Example skill — delete me

This file exists so:
- New contributors can see the skill format in practice, not just described in `skills/README.md`.
- The validator has something to check on a fresh checkout (so teams see a green light immediately after copying the boilerplate).

## When to use

Never, in production. Treat this as a template: duplicate it, rename the folder, and replace the contents with real guidance.

## How to write your first real skill

1. Duplicate `.ai/skills/_example/` and rename the folder to your skill name (e.g. `create-components`).
2. Rewrite this file with prescriptive, step-by-step instructions for the task.
3. Update the `last_reviewed` date in the frontmatter to today.
4. Add a row to the skills index in `.ai/instructions.md` — a skill with no index entry is invisible to the agent.
5. Delete the `_example` folder and its index row.
